树状数组能解决的问题是线段树能解决的问题的子集，但树状数组的代码更简短

## 核心思想

利用二进制来高效管理数据
想象你把街道上的店铺分成一些特殊的小组：

- 第 1 家店自己是一个小组。
- 第 2 家店和第 1 家店组成一个小组。
- 第 3 家店自己是一个小组。
- 第 4 家店和前面三家店组成一个小组。
- ...依此类推。

这种分组看似杂乱，但它背后隐藏着一个巧妙的规律：**每个小组代表的长度都是 2 的整数次幂**。

- 第 1 家店：小组长度 $1=2^0$。
- 第 2 家店：小组长度 $2=2^1$。
- 第 3 家店：小组长度 $1=2^0$。
- 第 4 家店：小组长度 $4=2^2$。
- 第 6 家店：小组长度 $2=2^1$。

这个规律的数学表示就是：`lowbit(i)`，它等于 i 的二进制表示中最低位的 `1` 所对应的值。比如 i=6 (二进制 `110`)，`lowbit(6)` 就是 2 (二进制 `10`)。
每个小组的结尾就是 `i`，它管理的范围就是 `i - lowbit(i) + 1` 到 `i`

1. **查询前缀和（1 到 R 的总和）**：
    - 你想要知道前 `R` 家店的总销售额。
    - 你从 `R` 开始，找到它所在的小组（`C[R]`）。
    - 然后，你从 `R` 跳到 `R - lowbit(R)` 的位置，继续寻找下一个小组。
    - 你重复这个过程，直到跳到 0。
    - 把沿途经过的所有小组的总和加起来，就是你想要的结果。这个过程只需要 O(logN) 步。

2. **更新单点值（第 K 家店的销售额）**：
    - 当第 `K` 家店的销售额改变了，它的改变会影响它所参与的所有小组。
    - 你从 `K` 开始，找到它所属的第一个小组。
    - 然后，你从 `K` 跳到 `K + lowbit(K)` 的位置，找到下一个包含 `K` 的小组。
    - 你重复这个过程，一直到数组末尾。
    - 每次都更新对应小组的总和。这个过程也只需要 O(logN) 步。

## 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 10;
int c[N], n, m;

int lowbit(int x) {
    return x & -x;
}
void add(int x, int val) {
    while (x <= n) {
        c[x] += val;
        x += lowbit(x);
    }
}

int query(int x) {
    int res = 0;
    while (x) {
        res += c[x];
        x -= lowbit(x);
    }
    return res;
}



int main() {
    cin >> n >> m;//数组大小&操作次数
    for (int i = 1; i <= n; i++) {
        int val;
        cin >> val;
        add(i, val);
```
