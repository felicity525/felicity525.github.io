C++ STL

什么是STL🧐

STL即标准模板库（Standard Template Library），是C++标准库的一部分，提供了很多可直接使用的数据结构和算法。一般算法竞赛都支持STL库的使用，合理使用STL可以避免大量的造轮子工作，提高效率。

STL容器

迭代器

迭代器用于访问STL容器中的元素（类似指针
#include<iterator>
常用运算符

  • 自增++自减—迭代器前后移动1位
  • +或-整型前后移动指定位数
  • 解引用*用于获取或修改它指向的元素
  • 头迭代器begin()可以获得容器第一个元素的迭代器
  • 尾迭代器end()可以获得指向容器尾端的迭代器（“尾端”不是最后一个元素，可以视为最后一个元素的后继）
    示例：使用迭代器遍历vector 使用auto关键字简化迭代器类型 C++11之后支持

    • v[pos]返回下标为pos的引用

  • 末尾添加新元素a.push_back(m)
    • 平均时间复杂度为
    • 最坏时间复杂度（若超出当前分配内存的大小则重新分配内存并复制已有数据）

  • 判断数组是否为空a.empty()返回bool类型
  • 清空数组a.clear()

    priority_queue

    优先队列（默认为大顶堆，即队列顶部为最大的元素）：按照元素的优先级进行排序，（默认）每次出队操作总是弹出优先级最高的元素

  • 访问队列顶部元素pq.top() 返回优先级最高的元素的引用，（时间复杂度为）

    set

  • set、multiset和map通常采用红黑树实现，其插入、查看和删除操作的时间复杂度近似为
    类似数学中的集合，set中不会出现值相同的元素，默认从小到大排序
  • 头文件#include<set>
  • 创建set<int>s
  • 插入s.insert(x)

    • erase(first,last)删除迭代器在范围内的所有元素
  • 获取大小s.size()
  • 清空s.clear()
  • 查找
    • count(x)返回set内键为x的元素数量
    • find(x)返回set内键为x的元素的迭代器，若不存在该元素则返回end()
    • lower_bound(x)返回指向首个不小于x的元素的迭代器，若不存在则返回end()
    • upper_bound(x)返回指向首个大于x的元素的迭代器，若不存在则返回end()
  • 判空s.empty()
  • 自定义比较方式

    用于将两个变量关联在一起
  • 头文件#include<pair>
  • 初始化

    • 先定义后赋值pair<int,double>p1; p1.first = 1; p1.second = 0.1
    • 使用make_pair函数pair<int,double>p2 = make_pair(1,0.1)

  • 与其他容器组合使用vector<pair<int,int>>

    STL算法

    STL提供了很多实现算法的模板函数，基本都包含在<algorithm>头文件中。
  • find： find(v.begin(),v.end(),value)顺序查找值为value的元素
  • swap：交换两个变量的值swap(a,b)
  • reverse：翻转数组、字符串,时间复杂度
    • reverse(v.begin(),v.end())反转整个容器
    • reverse(v.begin() + 1, v.begin() + 3)对容器下标部分进行反转
  • sort：排序，默认将元素从小到大排序，也可以通过自定义比较函数cmp实现复杂比较,时间复杂度
    • sort(v.begin(),v.end(),cmp)对整个容器排序
    • sort(v.begin() + 1, v.begin() + 3, cmp)对容器下标部分进行排序
  • merge
    将两个有序区间有序合并到第三个序列的迭代器上
  • max和min：max(a,b)
    求多个元素的最大值或最小值：max(a,max(b,c))对于c++11之后max({a,b,c})

    常见排序算法

    冒泡排序

    算法步骤（升序排序为例）
    1.从第一个元素开始，比较相邻的两个元素，若前一个元素更大，则交换两个元素；
    2.对每一对相邻元素重复上述步骤，直到容器末尾。此时最大的元素已被冒泡到容器末尾
    3.忽略已排序的最后一个元素，重复上述步骤，直至排序完成

时间复杂度 （两层for循环遍历数组）对于大规模数据效率较低
空间复杂度 (原地排序不需要额外的存储）

选择排序

算法步骤（升序排序为例）
1.将容器分为已排序部分和未排序部分（初始时已排序部分为空）
2.在未排序部分中找到最小的元素，与未排序部分的第一个元素交换位置
3.将未排序部分的起始位置向后移动一位，扩大已排序部分的范围
4.重复上述步骤直至未排序部分为空

时间复杂度（两层for循环遍历数组）对于大规模数据效率较低
空间复杂度 (原地排序不需要额外的存储）

插入排序

算法步骤（升序排序为例）
1.将容器分为已排序部分和未排序部分（初始时已排序部分只包含第一个元素）
2.在未排序部分中取出第一个元素，将该元素与已排序部分从后往前依次比较，找到合适的位置插入
3.重复上述步骤直至未排序部分为空

时间复杂度：
最坏原列表完全逆序，每次插入都需要移动所有已排序的元素
最好原列表有序，只需遍历一次列表
空间复杂度 (原地排序不需要额外的存储）

归并排序

算法步骤（升序排序为例）
1.将容器对半分为两个子数组
2.递归地对每个子数组进行排序
3.将两个已排序的子数组合并为一个有序数组
合并操作
1.申请空间存放待合并的数组
2.设定两个指针，最初分别指向两个待合并数组的起始位置
3.比较两个指针所指向的元素，选择较小的放入原数组，并移动指针到下一位置
4.重复步骤3直至某一指针到达数组末尾
5.将未到达末尾的数组的剩余元素复制到原数组

时间复杂度
分解：每次对半分，需要执行次
合并：
空间复杂度（合并操作需要额外空间来存储）

快速排序

算法步骤（升序排序为例）
1.选择一个元素作为基准
2.将所有小于基准元素的元素放在基准的左侧，所有大于基准元素的元素都放在基准的右侧
3.递归地对基准元素左侧和右侧的子数组进行快速排序

时间复杂度
分解：每次对半分，需要执行次
合并：
空间复杂度（原地排序，递归调用函数需要额外的空间）

希尔排序

对普通插入排序的优化
算法步骤（升序排序为例）
1.选择一个增量序列（常用n/2,n/4,...1)，用于将数组分成若干子数组
2.按照增量序列分组后，对每个子列表进行插入排序
3.逐步缩小增量，重复上述步骤直到增量为1
4.当增量为1时，对整个数组进行一次插入排序
示例：
初始数组 8，7，6，5，4，3，2，1
增量为4，分成4个子数组8,4；7,3；6,2；5,1各自插入排序后4,8 ; 3,7 ; 2,6 ; 1,5
此时数组为4,3,2,1,8,7,6,5
增量取2，分为4,2,8,6和3,1,7,5各自排序后2,4,6,8和1,3,5,7
此时2,1,4,3,6,5,8,7
增量为1，进行一次插入排序后得到有序序列1,2,3,4,5,6,7,8

时间复杂度取决于增量序列的选择，平均在到之间
空间复杂度（合并操作需要额外空间来存储）
